凌晨两点，我盯着终端输出发呆 — du -sh ~/projects 返回的结果是 87GB。我的 MacBook 只有 256GB 硬盘。

困境

作为一名全栈开发者，我的工作流离不开 Node.js。前后端项目、工具库、Demo 练习，日积月累，本地竟然有 60 多个项目文件夹。

问题不是项目文件本身，而是那些看似无害的 node_modules。

我原以为每个项目的依赖顶多几百 MB，直到那天深夜，我准备提交一个紧急修复，却发现：

"存储空间不足，无法保存此文件"

系统弹出的提示像一盆冷水。我打开系统存储管理，看到"系统数据"占了 62GB，完全不知道里面是什么。

我试着手动清理，但发现：

  • 有些 node_modules 嵌套在 node_modules 里，根本找不到
  • npm cache 偷偷占了 8GB
  • 有些项目停用多年，依赖包早就该清理了
  • 用 du -sh */ 一个个查，太慢了，而且看不出可视化分布

最崩溃的是：我花了 3 个小时手动删除了一些目录，结果重启后发现 —— 空间几乎没释放多少。

转折

就在我准备重装系统的时候，想起了前几天在公众号「诺果AI」看到的工具 —— DiskBot。

当时只是随手关注领了个激活码，没想到今天真用上了。

安装后打开，第一眼就被它的 TreeMap 可视化 震撼到了：

一个巨大的矩形树图，每个色块代表一个文件夹，块越大占的空间越多。我的 ~/projects 文件夹，赫然占据了整个图的一半面积。

我直接点击 projects 区块，下钻进去：

  • project-alpha/node_modules — 3.2GB
  • project-beta/node_modules — 2.8GB
  • old-experiments/node_modules — 5.1GB（这个项目我两年没碰了！）
  • .npm-cache — 8.4GB

原来如此！ 我之前手动删的都是些小文件，根本没碰到真正的大头。

实战

第一步：用 TreeMap 定位"重灾区"

DiskBot 的 TreeMap 让我一眼看清了空间分布。我只需要点击色块就能下钻，比 du -sh 快了 10 倍不止。

很快我发现： - 有 12 个项目 的 node_modules 超过 1GB - .npm 缓存目录竟然有 8.4GB - ~/Library/Application Support/npm-cache 还有 3.2GB

第二步：智能检测重复依赖

DiskBot 的 智能文件发现 功能，自动列出了我硬盘上的： - 大文件 Top 200（超过 100MB 的文件） - 旧文件（超过 365 天未修改） - 重复文件（基于 SHA256 哈希比对）

我发现有些 npm 包在不同项目里重复了 十几份，比如： - react@17.0.2 重复了 23 次 - lodash 重复了 31 次 - @types/node 重复了 18 次

DiskBot 的做法: 重复文件检测使用文件大小 + SHA256 双重校验，精准找出重复文件，告诉你到底浪费了多少空间。

第三步：一键清理缓存

DiskBot 的 缓存检测 功能，自动识别了 16+ 种常见缓存，包括： - npm cache (~/.npm) - Yarn cache (~/.yarn) - CocoaPods cache (~/Library/Caches/CocoaPods) - 各种构建工具缓存

我勾选了 npm cache 和 旧项目依赖，点击"移至废纸篓"。

DiskBot 后台异步执行清理，UI 不卡顿，我可以继续查看其他目录。

效果

清理完成后，我看了下结果：

  • 释放空间: 37.6GB
  • 删除文件数: 284,391 个
  • npm cache 清理: 8.4GB
  • 旧项目 node_modules 清理: 22.3GB
  • 重复文件清理: 6.9GB

最重要的是，系统"系统数据"从 62GB 降到了 28GB，终于不用再担心存储空间不足了。

我现在养成了习惯：每周用 DiskBot 扫描一次，及时发现新增的大文件和缓存。

复盘与建议

回想这次经历，有几个教训值得分享：

  1. 不要相信"系统数据"：macOS 的存储分析只能看到模糊分类，无法定位具体大文件。用 DiskBot 的 TreeMap，文件级粒度一目了然。

  2. 缓存是隐形杀手：npm、Yarn、CocoaPods 等工具的缓存，日积月累能吃掉几十 GB。定期清理非常必要。

  3. 可视化比命令行高效：之前用 du -sh 查，花 3 小时没解决问题。用 DiskBot 的可视化界面，10 分钟就搞定。

  4. 重复文件很浪费：同一个依赖包在不同项目里重复十几份，这是很多开发者忽略的存储浪费。

给程序员朋友的建议：别等到存储报警才清理。装个 DiskBot，每周花 5 分钟扫一扫，保持硬盘清爽，工作效率也能提升。
