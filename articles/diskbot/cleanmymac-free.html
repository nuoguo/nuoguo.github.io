每年花 300 块续费 CleanMyMac，你算过这笔账吗？

Mac 清理工具：一个被高估的市场

如果你是 Mac 用户，大概率遇到过那个令人窒息的弹窗——"您的磁盘空间不足"。

这时候你会做什么？打开 App Store 搜"磁盘清理"，然后看到一排标价 $9.99 到 $49.99 的应用。CleanMyMac X 是最有名的那个——订阅制每年 $39.95（约 290 元），买断制 $89.95（约 650 元）。

DaisyDisk 相对便宜，也要 $9.99。OmniDiskSweeper 倒是免费，但已经多年没更新了。

你有没有想过：清理自己电脑上的文件，为什么要花钱？

商业清理工具的"秘密"

我用了两年 CleanMyMac。说实话，它的核心功能就三个：

  1. 扫描磁盘——告诉你哪些文件占了空间
  2. 识别缓存——找到可以安全删除的临时文件
  3. 一键清理——帮你删掉这些文件

这三个功能值 300 块吗？

更关键的是，CleanMyMac 的"智能清理"有时候过于激进。有用户反馈它会误删应用配置文件，导致软件需要重新设置。而对开发者来说，它的缓存清理远远不够——不认识 node_modules、DerivedData、Docker 镜像这些开发者特有的"空间黑洞"。

免费替代：DiskBot 做对了什么

最近发现一个叫 DiskBot 的免费工具，试用之后有点意外。

DiskBot 的核心理念：用可视化让你看清磁盘空间到底去了哪里，然后自己决定删什么。不替你做决定，但帮你做好决定。

TreeMap 可视化：一眼看见空间黑洞

CleanMyMac 给你一个文件列表。DiskBot 给你一张矩形树图——每个色块代表一个文件或文件夹，面积越大占用越多。

这种交互让你直觉性地发现问题。不需要盯着文件大小数字，一眼就知道是 ~/Library/Developer 还是 ~/Downloads 在疯狂吃空间。

开发者友好的缓存清理

这是让我最满意的部分。DiskBot 内置了 16 种以上的缓存检测规则，覆盖：

  • Xcode DerivedData（经常 20GB+，很多开发者根本不知道）
  • npm / yarn 缓存
  • pip / conda 缓存
  • Docker 镜像和容器
  • Homebrew 下载缓存
  • 浏览器缓存（Chrome、Safari、Firefox）

每种缓存都标注了安全等级，让你知道删除后有什么影响。

重复文件检测

基于文件大小 + SHA256 哈希双重校验，不会误判"同名但内容不同"的文件。我扫描之后发现自己有 8.3GB 的重复 PDF 和压缩包——这些 CleanMyMac 从来没帮我找到过。
