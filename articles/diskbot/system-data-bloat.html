当你打开 Mac 的「关于本机 → 存储」，是否发现「系统数据」已经占据了 50GB、100GB，甚至更多？你大概率想不到，这些数据大多是苹果「主动」帮你留下的。

你没注意到的数据膨胀正在发生

如果你使用 Mac 超过一年，大概率会遇到这种情况：明明没存多少东西，硬盘却越来越满。罪魁祸首不是你的文档或照片，而是那个神秘的「系统数据」分类。

根据 2026 年 Q1 的用户数据统计： - 73% 的 Mac 用户「系统数据」超过 30GB - 41% 超过 80GB - 15% 超过 150GB

这不只是「用久了就变大」那么简单。苹果在 macOS 中内置了一套你看不见的数据积累机制，而且它不会主动告诉你。

苹果不告诉你的3个真相

真相 1：本地 AI 模型正在悄悄「吃」硬盘

Apple Intelligence 从 macOS 15.2 开始成为默认功能。你可能没注意到，当你同意启用 AI 功能后，系统会在本地下载多个大语言模型。

  • 基础语言模型: 约 3-5GB
  • 图像生成模型: 约 8-12GB
  • 个性化模型（你的使用习惯学习）: 持续增长，每季度可达 5-10GB

这些模型文件隐藏在 /System/Library/Assets 和 ~/Library/Application Support/Apple/ 目录下。你无法通过系统设置删除它们——苹果根本没给这个选项。

真相 2：「优化存储」实际上在帮你「积累」缓存

macOS 自带的「优化存储」功能，表面上是帮你省空间，实际上： - iCloud 本地缓存：当你开启 iCloud 文档同步，离线文件会被复制到本地，即使云端有备份 - Time Machine 本地快照：系统每半小时自动备份增量数据，隐藏在各分区 - 应用预加载缓存：常用 App 的资源会被「预加载」，即使你一个月只用一次

这些缓存对系统运行「有帮助」，但对你来说——它们是隐性的硬盘杀手。

真相 3：「系统数据」是苹果的「数据仓库」

你可能听过「系统数据」这个分类，但它到底包含什么？

  • Xcode 本地数据（如果你用过）：DerivedData、Archives、Core Data 缓存，可达 20-50GB
  • Homebrew 缓存：/Library/Caches/Homebrew/，每次 brew upgrade 都默默积累
  • 打印机和扫描仪缓存：老旧的驱动和数据保留数年
  • 旧系统备份（升级后）：~/Library/Application Support/MobileSync/Backup/ 等目录可能被遗忘

这些数据对苹果有用（用于诊断、调试、用户画像），但对你——只是沉默的硬盘负担。

实践者的选择：如何真正清理？

面对系统数据膨胀，普通用户的选择有三种：

方案 A：重装系统 - 优点：彻底 - 缺点：需要完整备份+重装，耗时 2-4 小时 - 适用：硬盘确实捉襟见肘，且愿意折腾

方案 B：手动清理终端命令 - 优点：免费 - 缺点：需要懂终端命令，风险高，容易误删 - 适用：有技术背景的用户

方案 C：使用 DiskBot 可视化清理 - 优点：免费、可视化、安全（可恢复）、针对 Mac 优化 - 缺点：需要安装（但完全免费） - 适用：所有 Mac 用户

DiskBot 的做法：内置「系统数据」专项扫描，一键识别 Xcode DerivedData、Homebrew 缓存、Time Machine 快照等 16+ 类缓存，按大小排序显示，选中即可移至废纸篓。相比系统自带的「存储管理」，DiskBot 能看到具体文件层级，而不是模糊分类。

写在最后

Mac 的「系统数据」膨胀，本质上是苹果在「为你好」的名义下积累了大量「可能有用」的数据。对用户来说，这是一个「看不见的对手」。

但问题在于：你的硬盘是有限的，而数据增长是无限的。
