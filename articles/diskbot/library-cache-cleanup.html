读完本文，你将能够：识别 ~/Library 下 16 种缓存的安全等级，用 DiskBot 一键清理，释放 20-100GB 不等的磁盘空间。

目标

~/Library 是 macOS 的"幕后仓库"——系统、应用、编译工具的缓存全都藏在这里。

打开Finder，按下 Command + Shift + G，输入 ~/Library，你可能会看到几十个文件夹，而它们加起来往往占据了几十甚至上百 GB 的空间。

本文的目标：告诉你哪些能删、哪些不能碰，并用 DiskBot 把这个过程变得可视化、安全化。

准备工作

在开始之前，确保你：

  • 使用的是 macOS 14 (Sonoma) 及以上版本
  • 已经安装 DiskBot v1.1.0 并激活
  • 已经用 Time Machine 或外接硬盘做了完整备份

DiskBot 的缓存安全等级说明： - 🔴 危险：删了会丢数据/崩溃 - 🟡 注意：可以删，但下次可能重新生成 - 🟢 安全：删了不影响任何功能

预期结果: 缓存检测页面列出所有 16 种缓存，显示各自的文件夹大小和风险标签。

第 2 步：解读风险等级，决定清理范围

DiskBot 的风险等级基于以下判断逻辑：

🔴 危险（不清理） - 数据库文件（.sqlite, .db）如果正在被应用访问，删除会导致数据丢失 - ~/Library/Application Support 下的内容：包含应用配置和用户数据，删了应用会重置

🟡 注意（谨慎清理） - Xcode Archives：包含你手动打包的 .ipa/.xcarchive，删了需要重新 archive - Docker 数据：删除后所有镜像和容器消失，如果你需要保留镜像，跳过此项 - Maven 本地仓库（~/.m2/repository）：删除后所有依赖需要重新下载

🟢 安全（放心清理） - DerivedData：每次 Xcode Build 自动重新生成，删了不影响项目 - npm/pip 缓存：只影响首次下载速度，不影响已安装包 - 浏览器缓存：纯粹是网页资源，删了浏览器会自动重建

小技巧: 在 DiskBot 中点击缓存项，可以查看该文件夹的详细文件列表和大小分布。如果你不确定，先点进去看看再决定。

预期结果: 对照风险等级，在 DiskBot 中勾选你要清理的缓存类型（建议优先勾选所有 🟢 安全项）。

第 3 步：执行清理，等待完成

勾选完要清理的缓存后，点击右下角的 「移至废纸篓」 按钮。

DiskBot 会执行以下操作：

  1. 将选中文件移至系统废纸篓（不是永久删除）
  2. 实时显示清理进度条
  3. 每清理完一项，增量更新 UI，不会卡顿

注意：如果某个文件正在被系统或应用占用，DiskBot 会跳过该项目并在日志中标注原因，不会强制删除导致系统异常。

第 4 步：清空废纸篓，真正释放空间

DiskBot 清理的文件会进入系统废纸篓。如果确认没有问题，在 Finder 中 右键 → 清倒废纸篓，或者在终端执行：

# 永久删除废纸篓中的所有文件
rm -rf ~/.Trash/*

提示: 废纸篓清空后空间才会真正释放。如果你不确定，建议等 24 小时后再清空，留出反悔时间。

常见问题

Q: 清理 DerivedData 会影响我正在开发的项目吗？

A: 不会。DerivedData 是 Xcode 的编译缓存，包含每次 Build 的中间产物。删除后，Xcode 会自动重新编译项目。如果项目较大，第一次重新 Build 可能需要几分钟，但这完全不影响代码本身。

Q: Docker 数据删了之后，之前的镜像还在吗？

A: 不在。如果你需要保留镜像，先用 docker save 导出备份，或者在 DiskBot 中跳过 Docker 这一项。运行中的容器不受影响（因为容器是进程，不是文件）。

Q: 清理后应用变慢了怎么办？

A: 浏览器等应用的首次访问可能会稍慢（需要重新缓存），这是正常现象。npm/pip 包管理器首次安装时会重新下载依赖。如果一周内恢复正常，无需担心。

Q: ~/Library/Logs 清理了会影响系统日志排查吗？

A: 不会。系统日志会实时写入，当前日志不受影响。清理旧的日志文件（通常是 .log.gz 压缩包）对诊断没有任何影响。

总结

~/Library 不是洪水猛兽，但确实是磁盘空间的"隐形大户"。通过 DiskBot 的缓存检测功能，你可以：

  • 🔍 可视化查看每种缓存的实际占用
  • ⚠️ 分级标注危险程度，避免误删重要数据
  • 🚀 一键批量清理，不用记命令、不用手动翻目录

下次当你的 Mac 弹出"存储空间不足"的提示时，别急着买新硬盘——先打开 DiskBot，看看 ~/Library 里藏着多少可以释放的空间。
